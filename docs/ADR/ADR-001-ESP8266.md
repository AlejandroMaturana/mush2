# ADR-001: Selección de la placa Wemos D1 Mini como nodo de telemetría

**Fecha**: 2026-06-10 (actualizado 2026-06-14)
**Estado**: Aceptado

## Contexto
El sistema requiere un microcontrolador con conectividad WiFi integrada para actuar como nodo de adquisición y telemetría ambiental. Debe leer los sensores AHT21+ENS160 vía I²C, enviar datos por MQTT y HTTP (ThingSpeak), controlar actuadores mediante SSRs, y operar de forma continua. El presupuesto objetivo por nodo es inferior a 5 USD.

## Decisión
Usar la placa **Wemos D1 Mini** basada en el ESP8266EX (módulo ESP-12F).

## Motivos
1. **Factor de forma compacto**: La D1 Mini (34 × 26 mm) es ideal para instalaciones en espacios reducidos como cámaras de cultivo.
2. **WiFi 802.11 b/g/n integrado**: El ESP8266EX incorpora pila TCP/IP completa y soporte para WPA2.
3. **Conector micro-USB con regulador CH340G**: Alimentación y programación por el mismo puerto USB.
4. **GPIOs suficientes**: Para este proyecto se requieren 2 pines I²C (D1/GPIO5 para SCL, D2/GPIO4 para SDA) y 3 pines para SSRs (D5, D4, D6).
5. **Compatible con PlatformIO**: La placa se programa con el ecosistema Arduino/ESP8266.
6. **Coste unitario inferior a 4 USD**.
7. **Memoria Flash de 4 MB**: Permite OTA (Over-The-Air updates).

## Consecuencias
- La D1 Mini opera a 3.3 V en los GPIOs. Los sensores AHT21+ENS160 trabajan nativamente a 3.3 V.
- El pin A0 tiene un divisor de tensión incorporado que limita el rango a 0–3.2 V.
- El CH340G requiere instalación de drivers en Windows y macOS; en Linux funciona nativamente.
- La antena cerámica del ESP-12F tiene alcance limitado (~30 m en interior).

## Alternativas descartadas
- **LOLIN D1 R1**: Similar pero con footprint de Arduino Uno. Se descartó por ser más grande sin beneficio adicional.
- **ESP32 DevKit v1**: Más caro y con consumo superior. Capacidades adicionales innecesarias.
- **NodeMCU v3**: Factor de forma no estándar que dificulta montaje compacto.

## Detalle técnico

### Mapeo de pines utilizado
| Función       | Pin D1 Mini | GPIO ESP8266 | Nota                        |
|---------------|-------------|--------------|-----------------------------|
| I²C SDA       | D2          | GPIO4        | Sensor AHT21+ENS160         |
| I²C SCL       | D1          | GPIO5        | Sensor AHT21+ENS160         |
| LED onboard   | D4          | GPIO2        | Uso exclusivo para debug    |
| SSR Canal 1   | D5          | GPIO14       | Ventilación                 |
| SSR Canal 2   | D4          | GPIO2        | Calefacción                 |
| SSR Canal 3   | D6          | GPIO12       | Humidificación              |

### Configuración en PlatformIO
```ini
[env:d1_mini]
platform = espressif8266
board = d1_mini
framework = arduino
monitor_speed = 115200
board_build.flash_mode = dout
```

### Intervalos de operación
- Lectura de sensores: cada 10 segundos (`SENSOR_INTERVAL`)
- Envío a ThingSpeak: cada 20 segundos (`TS_INTERVAL`)
